## Pipeline overview

```
data/calibrated_2019/<NN>_*_2019.md          (canonical source-of-truth, hand-written)
         │
         ▼ derive_calibrated_2019_csvs.py    (common skeleton parser)
data/calibrated_2019/csv/<NN>_*.csv          (long marginals + wide joints + cal target)
         │
         ▼ validate_calibrated_2019.py       (5 gates per AC)
data/calibrated_2019/audit/<NN>_gate_report.md
         │
         ▼ build_ac_verifier_configs.py      (CSVs → starter verifier configs)
simulations/wb_2021_ac<NN>/structures/{axes,joints,aggregate_targets}.json
         │
         ▼ scripts/per_ac/<NN>_extensions.py (optional, AC-specific tweaks)
         │
         ▼ kaisim/pipeline/verifiers/        (existing infra: marginal/joint/aggregate/composite)
                                              consumes the per-AC config and a sampled
                                              population, emits GapReport with chi-square score
```

extension scripts handle local divergences.

## Hard failures requiring MD-level fixes

### AC 064 — caste partition sums to 93.90% (off by 6.1pp)
- **Root cause**: `caste,UC bhadralok` row stores `10.0` (= % of Hindu pop)
  rather than `5.5` (= % of total pop). The MD's own arithmetic-check row
  inside §C.2 acknowledges this: `19.5 SC + 2.2 ST + 5.5 UC-Hindu + 8.0
  OBC-Hindu + 10.8 Other-Hindu + 42 Muslim + 1.4 Other`. The leaf-row
  pct is wrong.
- **Fix**: edit `064_murshidabad_2019.md` §C.2 to change UC-bhadralok
  pct from 10.0 → 5.5. Re-derive + re-validate.

### AC 210 — caste partition sums to 98.34% (off by 1.7pp)
- **Root cause**: minor rounding/imputation error somewhere in §C.2.
- **Fix**: TBD — review `210_nandigram_2019.md` §C.2 for the gap.

## Universal soft warnings

### Gate 4 (vote calibration) — soft warns on every AC except 095
- **Root cause**: `Σ P(party|religion) × P(religion)` recovers only
  Hindu/Muslim component; combined buckets in §E (e.g., `CPI(M)+INC` for
  095, multi-candidate sums for others) require explicit re-aggregation
  the validator doesn't yet do.
- **Status**: deliberate v0 limitation. Marginal numerical recovery is
  still meaningful as a sanity check; tightening this gate is a v1 task.

### Gate 2 (D.1 lang recovery) — soft warns on 5 ACs (003, 011, 123, 159, 222)
- **Root cause**: AC-local language palettes (Sadri, Santali, Mundari,
  Kurmali, etc.) — when the marginal C.5 has a fine-grained language
  list but the joint D.1 pools some into `Other`, the recovery diverges
  by 5–15pp on the long tail.
- **Status**: not a data error per se, just structural — recovery is
  approximate when joint aggregation differs from marginal aggregation.

### Gate 3 (population calibration vs ECI roll)
- **Skipped for all 10 ACs**: ECI 2019 final electoral roll PDFs not
  fetched. Same gap as §7 of `methodology_2019.md`.
