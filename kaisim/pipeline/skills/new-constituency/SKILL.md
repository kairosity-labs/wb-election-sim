# Kaisim — New Constituency Scaffolding

You are scaffolding a brand-new Kaisim simulation. Each simulation lives at
`kaisim/simulations/<sim_name>/` and is region+election specific.

## Step 1 — Determine sim_name

Pattern: `<region>_<year>_<scope>` (e.g., `wb_2021_ac095`, `up_2027_ac042`,
`maharashtra_2024_state`). Confirm with the user before creating directories.

## Step 2 — Scaffold directory tree

```bash
SIM=<sim_name>
mkdir -p kaisim/simulations/$SIM/{structures,derived,prompts,news,persona_configs,personas,simulation_configs,experiments,runs}
touch kaisim/simulations/$SIM/derived/__init__.py
touch kaisim/simulations/$SIM/__init__.py
```

## Step 3 — Files the user MUST provide

  - `news/events_<period>.yaml` — voter-salient events with tags + valence
    (see `kaisim/simulations/wb_2021_ac095/news/events_2019_2024.yaml` for template)
  - `structures/axes.json` + `structures/joints.json` — generated by a
    `build_structures.py` script that the user writes to translate their
    region's CSVs (see WB AC-95 template)
  - `derived/persona_tags.py` — region-specific tag derivation function
    (must define `derive_tags(persona) -> set[str]` and
    `derive_media_engagement(persona) -> float`)
  - `prompts/system_anchors.md` — narrative facts for persona generation
  - `news/persona_context_pre<year>.md` — opinion climate at calibration year

## Step 4 — Configs to copy + edit

  - `persona_configs/baseline_rule.yaml` — copy from WB template, adjust:
      - `persona_set.name`
      - `vote.parties` and `vote.blending.inputs[].table` (to match your joints)
      - `vote.welfare_overlap.precedence` (your region's welfare schemes)
  - `persona_configs/default.yaml` — same, plus:
      - `llm.provider` / `llm.model` of your choice
      - `prompts.anchors`, `prompts.news_context` paths
  - `simulation_configs/rule_based.yaml` — copy from WB template, adjust:
      - `clock.start_date` / `clock.end_date` to your period
      - `targeting.config.weights` if defaults don't suit your region
      - `ground_truth` for post-run analysis

## Step 5 — Validate end-to-end

```bash
# 1. Build structures (parses CSVs → axes.json + joints.json)
python kaisim/simulations/$SIM/build_structures.py

# 2. Generate baseline (rule-based) persona set — sanity check
python kaisim/simulations/$SIM/generate.py baseline_rule

# 3. Generate LLM persona set
python kaisim/simulations/$SIM/generate.py default

# 4. Run simulation
python kaisim/simulations/$SIM/run_simulation.py rule_based

# 5. Analyze
python -c "
import sys; sys.path.insert(0, 'kaisim')
from pathlib import Path
from pipeline.simulation.belief_analysis import run
run(Path('kaisim/simulations/$SIM/runs/<latest_run_dir>'))
"
```

## Common pitfalls

  - **CSV labels vs joint keys**: your `build_structures.py` MUST normalize
    short canonical codes consistently (see WB's `CANONICAL_CODES` dict)
  - **Voter-only filter**: most marginals are voter-population (18+); filter
    age cohorts to 18+ in build_structures and renormalize
  - **Verify_condition for occupation/class_of_worker**: these target
    workers only — declare `verify_condition` so the verifier filters
  - **Joint compound-bucket joints**: mark these `use: "sampling_only"`
    so verifier doesn't trip on them
